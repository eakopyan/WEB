Introduction

Les séances précédentes vous ont familiarisé avec :

Ce sujet a pour objectif d’assembler ces éléments. Le scénario choisi est un wiki.

De nombreuses portions de code vous seront fournies afin de vous fournir une structure applicative homogène.

Installation

Le projet utilise Flask et SQLAlchemy/Core comme vu lors des séances précédentes. Nous utiliserons une base de données SQLite.

Le rendu Markdown nécessite une nouvelle dépendance : markdown qui est disponible via easy_install. Installez-là.

Nouvelles fonctionnalités

Le wiki étant désormais fonctionnel, vous pouvez ajouter les fonctionnalités suivantes.

Suppression de page

Facile !

Index de pages

L’URL /pages-index doit faire le rendu d’un document HTML qui contient une liste des pages du wiki. Pour chaque page du wiki, un élément de liste est un lien vers la page :

  • Main

  • Hello

  • BonsPlan

  • ProjetX

  • (…)

Aperçu Markdown à l'édition

Proposez un aperçu du rendu Markdown du texte avant qu’une page ne soit sauvegardée.

Le rendu se fera coté serveur via une requête Ajax asynchrone. Vous pourrez soit proposer un rendu au fil de la frappe, soit un rendu périodique, soit un rendu à la demande.

Commentaires

Ajoutez une fonctionnalité de commentaires pour chaque page, similaire à Wikipedia. Dit autrement, les commentaires d’une page sont un document Markdown.

Historique de pages

Chaque révision de page doit être stockée, ce qui permet d’en parcourir les différentes versions.

Vous pouvez aussi implémenter une fonctionnalité pour restaurer une version donnée.
