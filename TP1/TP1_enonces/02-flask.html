Introduction

Ce sujet a pour but de vous faire découvrir le framework web Flask.

Flask est souvent qualifié de "micro-framework" dans le sens où il propose un jeu restreint mais suffisant de fonctionnalités pour développer une application. Il ne vous impose pas de choix pour des services comme l’accès aux bases de données.

Une application Flask est un programme Python qui lance directement un serveur web de développement. Pour la mise en production d’une application, on passera plutôt par un serveur HTTP frontal comme Apache HTTPD, Lighttpd ou Nginx.

Gardez sous la main les liens utiles vers la documentation :

  • Documentation de Flask

  • API complète de Flask

  • Jinja2, le moteur de templates par défaut

Hello world

Commençons avec le programme suivant :

hello.py
from flask import Flask
app = Flask(__name__)

@app.route('/')       (1)
def hey():
  return 'Coucou !'   (2)

if __name__ == '__main__':
  app.run(debug=True) (3)
1 Gère les routes /.
2 Renvoie directement ce contenu.
3 debug=True offre des informations détaillées et le rechargement automatique de code.

Lancez-le avec la commande python hello.py, puis ouvrez un navigateur web sur http://localhost:5000/

  1. Changez la chaîne renvoyée par la fonction hey, puis rechargez la page dans votre navigateur. Que voyez-vous ?

  2. En ligne de commande, utilisez Curl pour voir l'échange HTTP : curl -v http://localhost:5000/.

  3. Ouvrez l’URL http://localhost:5000/plop. Que voyez-vous ?

  4. Ajoutez une fonction qui renvoie "Stanford" sur la route /excellence.

On peut déclarer plusieurs décorations @app.route(...) sur une même fonction afin de gérer plusieurs routes.

Logging

Votre code Python peut avoir besoin de tracer des évènements comme des erreurs ou des informations de debug.

La référence app obtenue par la création d’un objet Flask possède un objet pour les logs : app.logger. Il possède 3 méthodes debug(string), warning(string) et error(string).

Insérez des logs pour les évènements suivants :

  1. un message de debug pour le lancement de l’application,

  2. un message de warning pour la route /,

  3. un message d'erreur pour la route /excellence.

Routage

Variables

Il est possible de spécifier des variables dans les spécifications de routage :

@app.route('/user/<name>')
def user(name):
  return 'User: %s' % name

@app.route('/profile/<int:uid>/<action>')
def profile(uid, action):
  return 'UID: %d / Action: %s' % (uid, action)

Une variable se spécifie dans un fragment d’URL par <nom>, et il faut que la fonction gérant la route prenne un paramètre du même nom.

Par défaut, le type d’une variable sera une chaîne de caractères. Il est possible de spécifier un convertisseur : int, float ou path (chaîne qui accepte des /).

  1. Ajoutez ces fonctions dans votre programme.

  2. Testez différentes URLs.

  3. Que se passe t-il avec l’URL /profile/foo/bar ?

Méthodes HTTP

Par défaut, une route ne gère que les requêtes HTTP GET.

Il est possible de spécifier quelles sont les méthodes acceptées :

from flask import Flask, request    (1)
app = Flask(__name__)

# (...)

@app.route('/plop', methods=['GET', 'POST'])
def plop():
  if request.method == 'GET':       (2)
    return "Ceci fut un HTTP GET"
  else:
    return "Ceci fut un HTTP POST"
1 Important pour pouvoir utiliser request.
2 request contient les informations sur la requête HTTP en cours.

Vous pouvez tester le comportement avec Curl :

$ curl http://localhost:5000/plop
(...)
$ curl -X POST http://localhost:5000/plop
(...)

Générer des routes

Il est souvent utile de générer des routes plutôt que de les désigner en dur dans le code.

Vous pouvez utiliser url_for à cette fin :

from flask import Flask, request, url_for

# (...)

@app.route('/show-routes')
def show_routes():
  routes = ''
  with app.test_request_context():
    routes = routes + url_for('excellence') + '\n'
    routes = routes + url_for('plop') + '\n'
    routes = routes + url_for('excellence', aeres=True) + '\n'
    routes = routes + url_for('profile', uid=1234, action='save') + '\n'
  return routes

Testez avec curl http://localhost:5000/show-routes.

Redirections et erreurs

Le module flask propose abort(code http) pour arrêter l’exécution, et définir un code de statut HTTP qui en principe signale une erreur (4xx, 5xx).

De même, il propose redirect(url) pour rediriger vers une URL.

Considérons les fonctions suivantes :

@app.route('/vortex')
def vortex():
  return redirect(url_for('excellence', vortex=1))

@app.route('/fail')
def fail():
  abort(500)

  1. Ajoutez les imports manquants en plus de ces fonctions (les sections précédentes doivent vous avoir mis sur le chemin …).

  2. Testez.

fail() retourne un code de statut 500. Nous pouvons personnaliser la gestion de codes de statut particuliers :

@app.errorhandler(500)
def internal_error(error):
  return "Wooops!"

  1. Ajoutez et testez.

  2. Ajoutez un gestionnaire d’erreur pour les codes de statut 404 ("not found").

Rendu HTML

Contenu statique

Une application web contient généralement du contenu statique : CSS, images, JavaScript, etc.

Flask sert le contenu de fichiers statiques s’ils se trouvent dans le répertoire static/.

Partons d’une nouvelle application Flask minimale :

app.py
from flask import Flask

app = Flask(__name__)

if __name__ == '__main__':
  app.run(debug=True)

  1. Ajoutez un fichier static/hello.txt et mettez-y quelques lignes de texte.

  2. Vérifiez que vous pouvez y accéder via une URL /static/hello.txt.

Vous pouvez faire des redirections vers du contenu statique :

@app.route('/hello.txt')
def hello():
  return redirect(url_for('static', filename='hello.txt'))

Templates

Flask utilise le moteur de rendu Jinja2. Il est principalement utilisé pour faire du rendu HTML, mais vous pouvez en réalité pour faire du rendu de n’importe quel format texte.

Flask suppose que les templates sont situés dans templates/.

La fonction render_template du module flask permet de faire un rendu comme dans l’exemple suivant :

@app.route('/truth')
def truth():
  return render_template('truth.html', title='The truth', truth=42)

Avec un template :

templates/truth.html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <title>{{ title }}</title>
</head>
<body>
  <h1>Hello !</h1>
  <p>
    Here is the truth: {{ truth }}
  </p>
</body>
</html>

Consultez la documentation Jinja2 sur les templates, et essayez le code suivant :

@app.route('/contacts')
def contacts():
  data = [
    {'name': 'Mr Bean', 'email': 'mrbean@outlook.com'},
    {'name': 'John Appleseed', 'email': 'john.appleseed@icloud.com'}
  ]
  return render_template('contacts.html', data=data)
templates/contacts.html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <title>Contacts</title>
</head>
<body>
  <h1>Contacts</h1>
  <ul>
  {% for contact in data %}
    <li>{{ contact['name'] }} ({{ contact.email }})</li>
  {% endfor %}
  </ul>
</body>
</html>

Objet request

L’objet request du module flask vous donne accès à de nombreuses propriétés dont :

  • method pour la méthode HTTP utilisée,

  • path pour le chemin utilisé,

  • args pour les arguments passés à l’URL (e.g., /foo?a=1 est lu via request.args.get('a', 'valeur par défaut')),

  • form donne accès aux valeus de formulaires,

  • files donne accès à des fichiers envoyés (voir http://flask.pocoo.org/docs/patterns/fileuploads/#uploading-files),

  • cookies pour les cookies du client.

Un mur

Assemblons tous ces éléments pour faire un mur :

posts = []

@app.route('/')
@app.route('/index')
def index():
  return render_template('index.html', posts=posts)

@app.route('/post', methods=['POST'])
def post():
  posts.append(request.form['post'])
  return redirect(url_for('index'))

Ceci est suffisant pour gérer une liste de posts à l’aide du template suivant :

templates/index.html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <title>Posts</title>
</head>
<body>

{% for post in posts %}
<p>{{ post }}</p>
{% endfor %}

<p>
  <form action="/post" method="POST">
    <input type="text" name="post">
    <input type="submit" value="Boom !">
  </form>
</p>

</body>
</html>

  1. Testez.

  2. Que se passe t-il quand vous relancez l’application ?

  3. Modifiez le template templates/index.html pour afficher "(aucun post)" quand la liste des postes est vide.

Sessions

Flask sait gérer des sessions coté serveur. Il propose un objet session qui se comporte comme un dictionnaire Python ordinaire.

L’implémentation par défaut maintient la session à l’aide de cookies renvoyés au client et signés par une clé qui n’est connue que coté serveur.

L’application d’exemple est la suivante :

app.py
from flask import *

app = Flask(__name__)
app.secret_key = 'iswuygdedgv{&75619892__01;;>..zzqwQIHQIWS'  (1)

@app.route('/')
def index():
  logged = 'logged' in session                                (2)
  if logged:
    txt = 'Bonjour %s !' % session['name']
  else:
    txt = 'Bonjour illustre inconnu !'
  return render_template('index.html', message=txt, logged=logged)

@app.route('/login', methods=['POST'])
def login():
  session['name'] = escape(request.form['name'])              (3)
  session['logged'] = True
  return redirect('/')

@app.route('/logout')
def logout():
  session.clear()
  return redirect('/')

if __name__ == '__main__':
  app.run(debug=True)
1 La clé utilisée pour signer les cookies. Celle-ci n’est pas recommendable …
2 in permet de tester si un dictionnaire a une entrée.
3 escape permet de se prémunir de certaines attaques si la valeur est passée telle quelle dans une base de donnée ou dans du HTML.
templates/index.html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <title>Sessions</title>
</head>
<body>

<p>{{ message }}</p>

{% if logged %}
<p>
  <a href="/logout">[logout]</a>
</p>
{% else %}
<form action="/login" method="POST">
  <p>
    ... mais qui êtes vous ?
    <input type="text" name="name">
    <input type="submit" value="Voilà !">
  </p>
</form>
{% endif %}

</body>
</html>
